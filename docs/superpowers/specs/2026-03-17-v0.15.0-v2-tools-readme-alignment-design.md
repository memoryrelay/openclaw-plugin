## 1. V2 Agent-Facing Tools

### New Tool Group

Add a `v2` group to `TOOL_GROUPS` (currently `Record<string, string[]>` at line 1569 of `index.ts`):

### Tool: `memory_store_async`

Store memory asynchronously via V2 API. Returns immediately (<50ms) with job ID. Background workers generate the embedding.

**Parameters:**
| Name | Type | Required | Description |
|------|------|----------|-------------|
| `content` | string | Yes | Memory content (1-50,000 chars) |
| `metadata` | object | No | Key-value metadata |
| `project` | string | No | Project slug (max 100 chars) |
| `importance` | number | No | 0.0-1.0, values >= 0.8 promote to hot tier |
| `tier` | string | No | "hot", "warm", or "cold" |

**Returns:** `{ id, status, job_id, estimated_completion_seconds }`

**Client method:** `storeAsync(content, metadata?, project?, importance?, tier?)`
**API endpoint:** `POST /v2/memories`

### Tool: `memory_status`

Check processing status of memory created via `memory_store_async`.

**Parameters:**
| Name | Type | Required | Description |
|------|------|----------|-------------|
| `memory_id` | string | Yes | Memory ID (UUID) |

**Returns:** `{ id, status, created_at, updated_at, error? }`
- Status values: `pending`, `processing`, `ready`, `failed`

**Client method:** `getMemoryStatus(memoryId)`
**API endpoint:** `GET /v2/memories/{id}/status`

### Tool: `context_build`

Build ranked context bundle with optional AI summarization.

**Parameters:**
| Name | Type | Required | Description |
|------|------|----------|-------------|
| `query` | string | Yes | What information is needed |
| `max_memories` | number | No | 1-100, default 20 |
| `max_tokens` | number | No | Token budget, 100-128000 |
| `ai_enhanced` | boolean | No | Enable AI summary, default false |
| `search_mode` | string | No | "semantic", "hybrid" (default), or "keyword" |
| `exclude_memory_ids` | string[] | No | IDs to exclude from results |

**Returns:** Ranked context with memories and optional summary

**Client method:** `buildContextV2(query, options?)`
**API endpoint:** `POST /v2/context`

## 2. Client Methods

Add 3 new methods to `MemoryRelayClient` class (after existing `promote()` method, ~line 937):

### `storeAsync()`

```typescript
async storeAsync(
  content: string,
  metadata?: Record<string, string>,
  project?: string,
  importance?: number,
  tier?: string,
): Promise<{ id: string; status: string; job_id: string; estimated_completion_seconds: number }> {
  const body: Record<string, unknown> = {
    content,
    agent_id: this.agentId,
  };
  if (metadata) body.metadata = metadata;
  if (project) body.project = project;
  if (importance != null) body.importance = importance;
  if (tier) body.tier = tier;
  return this.request("POST", "/v2/memories", body);
}
```

### `getMemoryStatus()`

```typescript
async getMemoryStatus(memoryId: string): Promise<{
  id: string;
  status: "pending" | "processing" | "ready" | "failed";
  created_at: string;
  updated_at: string;
  error?: string;
}> {
  return this.request("GET", `/v2/memories/${memoryId}/status`);
}
```

### `buildContextV2()`

```typescript
async buildContextV2(
  query: string,
  options?: {
    maxMemories?: number;
    maxTokens?: number;
    aiEnhanced?: boolean;
    searchMode?: "semantic" | "hybrid" | "keyword";
    excludeMemoryIds?: string[];
  },
): Promise<any> {
  const body: Record<string, unknown> = {
    query,
    agent_id: this.agentId,
  };
  if (options?.maxMemories != null) body.max_memories = options.maxMemories;
  if (options?.maxTokens != null) body.max_tokens = options.maxTokens;
  if (options?.aiEnhanced != null) body.ai_enhanced = options.aiEnhanced;
  if (options?.searchMode) body.search_mode = options.searchMode;
  if (options?.excludeMemoryIds) body.exclude_memory_ids = options.excludeMemoryIds;
  return this.request("POST", "/v2/context", body);
}
```

## 3. Direct Command: `/memory-context`

### `/memory-context <query>`

Build a ranked context bundle. Wraps `context_build` tool for human inspection.

**Registration:** `acceptsArgs: true`

**Arguments:**
- `<query>` — required, positional
- `--max-memories <n>` — default 20
- `--max-tokens <n>` — token budget
- `--ai-enhanced` — boolean flag, enable AI summary
- `--search-mode <mode>` — semantic/hybrid/keyword

**Output:** Formatted context bundle showing:
- Number of memories included
- Total tokens used
- Each memory: content preview (120 chars), relevance score, category
- AI summary (if `--ai-enhanced`)

**Client method:** `client.buildContextV2(query, options)`

This brings the total direct commands from 15 to 16.
