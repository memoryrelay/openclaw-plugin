## 1. New Direct Commands

All commands use `api.registerCommand()` (same pattern as existing 5 commands). Commands that accept arguments must set `acceptsArgs: true` in their registration; the raw argument string is accessed via `ctx.args`. Commands without arguments set `acceptsArgs: false` (or omit it, as false is the default). Arguments are parsed with a hand-rolled utility (no `commander` dependency).

### New Commands (10)

#### `/memory-search <query>`

Semantic memory search with filters. Wraps `client.search()`.

**Arguments:**
- `<query>` — required, search text (first positional argument)
- `--limit <n>` — max results (default: 10)
- `--project <slug>` — filter by project
- `--threshold <n>` — similarity threshold 0-1 (default: 0.3)

**Output:** Formatted list of matching memories with:
- Content preview (first 120 chars)
- Similarity score
- Category
- Timestamp
- Memory ID

**Client method:** `client.search(query, limit, threshold, { project })`

#### `/memory-validate`

Production readiness check. Runs 7 checks sequentially:

1. API connectivity (health endpoint reachable)
2. API health (services status is healthy)
3. Core tools registered (memory_store, memory_recall, memory_list present)
4. Auto-recall enabled
5. Auto-capture enabled
6. Memory count (can list memories successfully)
7. Agent ID configured (not empty/default)

**Output:**
- Per-check status line: `[PASS]` / `[FAIL]` / `[WARN]`
- Summary: `7/7 passed`
- Letter grade: A+ (7/7), A (6/7), B+ (5/7), B (4/7), F (<4)
- Production ready: Yes/No

**No arguments.**

#### `/memory-config`

Read-only display of current effective configuration.

**Output:** Formatted key-value pairs:
- API URL
- Agent ID
- Default Project
- Enabled Tools
- Auto-Recall (enabled/disabled)
- Auto-Capture (enabled/disabled + tier)
- Recall Limit
- Recall Threshold
- Exclude Channels
- Session Timeout
- Session Cleanup Interval
- Debug
- Verbose
- Max Log Entries

**No arguments.**

#### `/memory-sessions`

List active and recent sessions. Wraps `client.listSessions()`.

**Arguments:**
- `--limit <n>` — max results (default: 10)
- `--status <status>` — filter by status (e.g., "active", "ended"); shorthand `--active` is equivalent to `--status active`
- `--project <slug>` — filter by project

**Output:** Formatted table with:
- Session ID
- Status (active/ended)
- Start time
- Duration or "ongoing"
- Summary (first 80 chars)

**Client method:** `client.listSessions(limit, project, status)`

#### `/memory-decisions`

List architectural decision records. Wraps `client.listDecisions()`.

**Arguments:**
- `--limit <n>` — max results (default: 10)
- `--project <slug>` — filter by project
- `--status <status>` — filter by status (active/superseded)
- `--tags <tags>` — filter by tags (comma-separated)

**Output:** Formatted list with:
- Decision title
- Status (active/superseded)
- Date
- Rationale preview (first 100 chars)

**Client method:** `client.listDecisions(limit, project, status, tags)`

#### `/memory-patterns`

List or search reusable patterns. Wraps `client.searchPatterns()`.

**Arguments:**
- `<query>` — optional search text (passes `""` to API if omitted, which returns all patterns)
- `--limit <n>` — max results (default: 10)
- `--category <cat>` — filter by category
- `--project <slug>` — filter by project

**Output:** Formatted list with:
- Pattern name
- Category
- Description preview (first 100 chars)

**Client method:** `client.searchPatterns(query, category, project, limit)`

#### `/memory-entities`

List entities in the knowledge graph. Wraps `client.listEntities()`.

**Arguments:**
- `--limit <n>` — max results (default: 20)

**Output:** Formatted list with:
- Entity name
- Entity type
- Relationship count (if available)

**Client method:** `client.listEntities(limit)`

#### `/memory-projects`

List registered projects. Wraps `client.listProjects()`.

**Arguments:**
- `--limit <n>` — max results (default: 20)

**Output:** Formatted list with:
- Project slug
- Project name/description
- Memory count (if available)

**Client method:** `client.listProjects(limit)`

#### `/memory-agents`

List registered agents. Wraps `client.listAgents()`.

**Arguments:**
- `--limit <n>` — max results (default: 20)

**Output:** Formatted list with:
- Agent ID
- Agent name
- Description

**Client method:** `client.listAgents(limit)`

#### `/memory-forget <id>`

Delete a memory by ID. Wraps `client.delete()`.

**Arguments:**
- `<id>` — required, memory ID to delete

**Output:**
- Success: Shows memory content preview before confirming deletion: `Deleted memory <id>: "<content preview>"`
- Not found: `Memory <id> not found.`
- Failure: error message

**Note:** No interactive confirmation (SDK command model is request-response). The output includes the deleted content so the user can verify what was removed.

**Client method:** `client.delete(id)`

### Argument Parsing

Hand-rolled parser (no dependencies). Shared utility function consuming `ctx.args`:

```typescript
function parseCommandArgs(input: string): { positional: string[], flags: Record<string, string | boolean> }
```

- Input is `ctx.args` (raw string after command name, provided by SDK when `acceptsArgs: true`)
- Splits on whitespace, respecting quoted strings (double and single quotes)
- `--flag value` → `{ flag: "value" }` (next token consumed as value unless it starts with `--`)
- `--flag` followed by another `--` or end of input → `{ flag: true }`
- Non-flag tokens → positional array
- Empty/undefined input → `{ positional: [], flags: {} }`

**Commands requiring `acceptsArgs: true`:** memory-search, memory-sessions, memory-decisions, memory-patterns, memory-entities, memory-projects, memory-agents, memory-forget

**Commands with `acceptsArgs: false` (default):** memory-validate, memory-config

## 2. Config Schema Fix (`openclaw.plugin.json`)

### Changes

**`autoCapture`** — change from `{ type: "boolean", default: false }` to:

```json
{
  "oneOf": [
    { "type": "boolean" },
    {
      "type": "object",
      "properties": {
        "enabled": { "type": "boolean", "default": true },
        "tier": { "type": "string", "enum": ["off", "conservative", "smart", "aggressive"], "default": "smart" },
        "confirmFirst": { "type": "number", "default": 5, "minimum": 0 },
        "categories": {
          "type": "object",
          "properties": {
            "credentials": { "type": "boolean", "default": true },
            "preferences": { "type": "boolean", "default": true },
            "technical": { "type": "boolean", "default": true },
            "personal": { "type": "boolean", "default": false, "description": "Privacy: personal info requires confirmation" }
          }
        },
        "blocklist": {
          "type": "array",
          "items": { "type": "string" },
          "default": [],
          "description": "Regex patterns for content to never capture"
        }
      }
    }
  ],
  "default": true,
  "description": "Auto-capture configuration. Use true/false for simple toggle, or object for fine-grained control."
}
```

   - Table: all commands with description, arguments, and example

7. **Configuration Reference**
   - Full config table matching `openclaw.plugin.json` schema
   - Environment variables: `MEMORYRELAY_API_KEY`, `MEMORYRELAY_AGENT_ID`, `MEMORYRELAY_API_URL`, `MEMORYRELAY_DEFAULT_PROJECT`
